## üìÑ Updating Your CV

**Super easy!** Just replace the file:

1. Export your updated CV as PDF
2. Name it exactly: `CV_Azeez_Hamzat.pdf`
3. Replace the file in the `docs/` folder
4. Done! The download button automatically uses the new file

**Tips:**
- Keep filename the same for automatic linking
- Optimize PDF size (compress if > 5MB)
- Ensure PDF is not password-protected

## üî¨ Updating Research Content

### Adding a New Research Project

1. Open `research.html`
2. Find the section: `<!-- Work in Progress -->` or `<!-- Completed Projects -->`
3. Copy an existing `<article class="project-item">` block
4. Update the content:

```html
<article class="project-item">
    <div class="project-header">
        <h4>Your Project Title</h4>
        <span class="project-status in-progress">In Progress</span>
    </div>
    <p class="project-meta">
        <i class="fas fa-calendar"></i> 2025 | 
        <i class="fas fa-flask"></i> Your Method
    </p>
    <p class="project-description">
        Description of your project...
    </p>
    <div class="project-tags">
        <span class="tag">Tag 1</span>
        <span class="tag">Tag 2</span>
    </div>
</article>
```

### Updating Research Interests

1. Open `research.html`
2. Find: `<section class="research-interests">`
3. Edit the text within `<article class="interest-card">` blocks

## üìö Updating Publications

### Adding a New Publication

1. Open `publications.html`
2. Find the appropriate section (Preprints, Works in Progress, etc.)
3. Copy an existing `<article class="publication-item">` block
4. Update with your publication details:

```html
<article class="publication-item">
    <h3>Your Publication Title</h3>
    <p class="publication-meta">
        <span class="author">Hamzat, A.A.</span> (2025). 
        <em>Journal Name</em>. 
        DOI: <a href="https://doi.org/..." target="_blank">10.xxxx/xxxxx</a>
    </p>
    <p>Brief description or abstract...</p>
    <div class="card-tags">
        <span class="tag">Keyword 1</span>
        <span class="tag">Keyword 2</span>
    </div>
</article>
```

### Updating Publication Status

When a work-in-progress is published:
1. Move it from "Works in Progress" section to "Preprints" or appropriate section
2. Update the DOI link
3. Change any status badges

## üé® Changing Colors and Styles

### Changing Color Scheme

1. Open `css/variables.css`
2. Update these main colors:

```css
:root {
  --color-primary: #667eea;      /* Your main brand color */
  --color-accent: #764ba2;       /* Secondary brand color */
  --color-dark-blue: #2c3e50;    /* Dark text color */
}
```

3. Save and refresh browser

### Changing Fonts

1. Go to [Google Fonts](https://fonts.google.com)
2. Select your fonts
3. Copy the `<link>` code
4. Replace font links in all HTML `<head>` sections
5. Update font families in `css/variables.css`:

```css
--font-heading: 'YourFont', sans-serif;
--font-body: 'YourFont', sans-serif;
```

## üîç SEO Updates

### Meta Descriptions

Each page has meta descriptions. Update them if page content changes significantly:

```html
<meta name="description" content="Your updated description here">
```

Keep descriptions:
- 150-160 characters
- Include relevant keywords
- Accurately describe page content

### Updating Sitemap

When you add new pages or blog posts:

1. Open `sitemap.xml`
2. Add new `<url>` entry:

```xml
<url>
  <loc>https://azeezhamzat.com/your-new-page.html</loc>
  <lastmod>2025-10-15</lastmod>
  <changefreq>monthly</changefreq>
  <priority>0.8</priority>
</url>
```

3. Update `<lastmod>` dates for changed pages

### Updating RSS Feed

When you add blog posts:

1. Open `feed.xml`
2. Add new `<item>` at the top:

```xml
<item>
  <title>Your Post Title</title>
  <link>https://azeezhamzat.com/blog/post.html?id=your-post-id</link>
  <guid>https://azeezhamzat.com/blog/post.html?id=your-post-id</guid>
  <pubDate>Mon, 15 Oct 2025 00:00:00 GMT</pubDate>
  <description>Your post description...</description>
  <author>azeezhamzat@yahoo.com (Azeez Adewale Hamzat)</author>
  <category>Your Category</category>
</item>
```

## üö® Common Mistakes to Avoid

### 1. Breaking HTML
- Always close tags: `<div>...</div>`
- Match opening and closing tags
- Don't nest tags incorrectly

### 2. Case-Sensitive Filenames
- `Image.jpg` ‚â† `image.jpg`
- Use lowercase for consistency
- Match filenames exactly in HTML

### 3. Forgetting to Update Multiple Places
Some info appears in multiple files:
- Email address
- Social media links
- Institution name

Search all files and update everywhere!

### 4. Large Image Files
- Compress images before uploading
- Target < 200KB per image
- Use tools: TinyPNG, Squoosh, ImageOptim

### 5. Not Testing After Changes
Always test your changes before deploying:
- Open pages in browser
- Check console for errors (F12)
- Test on mobile view
